# 戰術系統實戰測試計畫

> 適用版本：戰術移動＋豁免抵擋系列（PR #120 ～ 本次）。
> 建議人力：ST ×1＋玩家 ×2（其中一位用手機測試更佳）。預估 40～60 分鐘。
> 開始前**所有人強制重新整理**（Ctrl+Shift+R / 手機清快取），確保不是舊版腳本。

---

## 0. 前置準備（5 分鐘）

- [ ] ST 開房，兩位玩家加入
- [ ] ST 建立 1 隻 BOSS（設好 防禦 DP／防禦附加／三豁免／多重行動 3 招以上，其中至少 1 招勾 AOE 並填 DP）
- [ ] 每位玩家各建 1 隻我方單位並部署；ST 另建 1 隻敵方小怪
- [ ] （若 Firebase 規則有另行部署）確認已重新部署 `database.rules.json`

| D1 | 玩家發起攻擊，結算模式選「豁免抵擋」 | 出現「目標豁免」選擇（反射/意志/強韌）|

| D3 | ST 展開完整計算公式 | 有「豁免抵擋」「目標豁免」段落，豁免骰數 = BOSS 三豁免設定值 |
| D4 | ST 開自動擲骰確認 | 廣播：「擲 30 顆 → 成功 X ＋ 附加 2｜目標反射豁免擲 N 顆 → Y 成功 ➡️ 總傷害 max(0, X+2−Y)」，血條正確 |

| D10 | 再切回防禦扣除模式攻擊一次 | 行為回到 C1（模式殘留檢查）|

## 已知限制（測試時不算 bug）

1. AOE「豁免抵擋」結算不支援「復原上一步」（直接傷害模式支援）。
2. 單體豁免模式下，BOSS 的「防禦附加成功」與「無視防禦」不參與（豁免是獨立對擲）。
3. 移動能量只在「戰鬥中」結算；部署、收回、非戰鬥移動不消耗。
4. 玩家看不到敵方單位的三豁免數值（只有 ST 端審核明細會顯示骰數）。
